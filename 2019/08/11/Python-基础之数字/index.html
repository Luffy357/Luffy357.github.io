Python 表达式操作符及其优先级

操作符 描述
x or y 逻辑或(只要一个条件为真即为真)
x and y 逻辑与(只有 x 为真，才会计算 y)
not x 逻辑非(取反操作)
x in y, x not in y 成员关系（可迭代对象、集合）
x is y, x is not y 对象实体测试
x < y, x <= y, x > y,
x >= y, x == y, x != y 大小比较，集合子集和超集值相等性操作符
x | y 位或，集合并集
x ^ y 位异或，集合对称差
x & y 位与，集合交集
x << y, x >> y 位移或右位 y 位
x + y, x - y 加法/合并， 减法，集合差集
x * y, x % y, x / y, x // y 乘法/重复, 余数/格式化，真除法，整除或floor除法
-x, +x 一元加减，识别
~x 按位求补(取反)
x ** y 幂运算
x[i] 索引
x[i:j:k] 分片
x(…) 调用(函数，方法，类及其他可调用的)
x.attr 属性引用
(…) 元组, 表达式, 生成器表达式
[…] 列表，列表解析
{…} 字典, 集合、集合和字典解析

1、操作符越靠后的优先级越高，在混合使用时，一定要注意优先级
2、同一行的表达式在组合时候通常从左到右组合(除了幂运算是从右向左组合的，比较运算是从左到右连接的)
3、如果用括号分组的子表达式，会超越 Python 的优先级规则。总是最先计算括号中的表达式
4、数字的混合类型计算，会将对象转换为其中最复杂的类型的进行计算。比如 整数加浮点数，会将整数转换为浮点数

也可以使用小数上下文管理器，设置临时精度。语句退出后精度又为初始值

with decimal.localcontext() as ctx:
