Python 表达式操作符及其优先级

操作符 描述
x or y 逻辑或(只要一个条件为真即为真)
x and y 逻辑与(只有 x 为真，才会计算 y)
not x 逻辑非(取反操作)
x in y, x not in y 成员关系（可迭代对象、集合）
x is y, x is not y 对象实体测试
x < y, x <= y, x > y,
x >= y, x == y, x != y 大小比较，集合子集和超集值相等性操作符
x | y 位或，集合并集
x ^ y 位异或，集合对称差
x & y 位与，集合交集
x << y, x >> y 位移或右位 y 位
x + y, x - y 加法/合并， 减法，集合差集
x * y, x % y, x / y, x // y 乘法/重复, 余数/格式化，真除法，整除或floor除法
-x, +x 一元加减，识别
~x 按位求补(取反)
x ** y 幂运算
x[i] 索引
x[i:j:k] 分片
x(…) 调用(函数，方法，类及其他可调用的)
x.attr 属性引用
(…) 元组, 表达式, 生成器表达式
[…] 列表，列表解析
{…} 字典, 集合、集合和字典解析

1、操作符越靠后的优先级越高，在混合使用时，一定要注意优先级
2、同一行的表达式在组合时候通常从左到右组合(除了幂运算是从右向左组合的，比较运算是从左到右连接的)
3、如果用括号分组的子表达式，会超越 Python 的优先级规则。总是最先计算括号中的表达式
4、数字的混合类型计算，会将对象转换为其中最复杂的类型的进行计算。比如 整数加浮点数，会将整数转换为浮点数

除法

10 / 4 # 真除法，不管操作数的类型，都返回包含任何余数的浮点结果
10 // 4 # Floor 除法，只截取掉余数，返回的类型根据操作数的类型。只要有个浮点数，结果为浮点数

// 、 floor 不管整数还是负数总是向下取整

floor 并没有真正意义上截断负数的结果

import math

math.floor(2.5)
math.floor(-2.5)

math.trunc 正数向下取整，负数向上取整

math.trunc(2.5)
math.trunc(-2.5)

十六进制、八进制和二进制

Python 默认用十进制，如下内置函数将十进制转换为其他进制

oct(64), hex(64), bin(64)

‘{0:o}, {1:x}, {2:b}’.format(64, 64, 64)

int 函数将一个数字字符串通过对应的进制转换为整数，第二个参数为进制

int(‘64’), int(‘100’, 8), int(‘40’, 16), int(‘1000000’, 2)

位操作

x = 1
x << 2 # (0001) 左移 2 位 (0100)
x | 2 # (0001 | 0010 = 0011)
x & 1 # (0001 & 0001 = 0001)

bit_lenght 可以得到二进制字符串的长度

(256).bit_length()
