Python 动态类型

发表于 2019-08-14 | 分类于 Python

动态类型是 Python 的灵活性的根源

在 Python 中，类型是在运行过程中自动决定的，而不是通过代码声明，这意味着没有必要事先声明变量、对象的类型

变量

变量是没有类型的也不会被类型约束，类型的概念只存在于对象中，而不是变量名中。
变量出现在表达式中马上会被该变量所引用的对象代替

变量 = 对象 的操作只是将变量与对象连接，对象是一块内存，有足够的空间去表示它们所代表的值。
这个值是什么类型，该对象就是什么类型。类型时属于对象的而不是变量。牢记：Python 中变量没有类型。

对象

每个对象都有两个标准的头部信息：一个类型标志符去标识这个对象的类型，一个引用计数器。

对象的垃圾回收机制

变量 x、y 都引用对象 3.1415 的时候。这个浮点数对象就有两个引用计数。当 x 变量指向别的对象时候，
3.1415 对象的引用计数减 1。 当 y 变量也指向别的对象时。3.1415 对象引用计数为零那么这个对象
的内存空间就会被自动回收。在程序中一个对象的引用计数为零，就会释放这个对象的内存空间。

共享引用

变量 a 指向对象 3 ，变量 b 又指向了变量 a ，实际上变量a、b 都指向对象 3。当变量 a 指向别的对象时候
变量 b 还是指向 3。

整数是不可变对象，因此没有方法在原处修改它，最一条语句其实变量 a 最后指向的是对象 5。可以
通过 id 函数来查看变量指向内存地址。会发现前一个和后一个变量 a 指向不是同一个内存空间地址。

在 Pthon 中变量总是一个指向对象的指针，而不是可改变的内存区域的标签：给一个变量赋一个新的值，
并不是替换了原始的对象，而是让这个变量去引用完全不同的一个对象。改变了变量的指向。

共享引用和在原处修改

变量 L1 指向了一新的对象，而变量 L2 还是指向原来的对象。

因为列表是可变类型，变量 L1、L2 同时指向一个对象。当 L1 对列表进行修改时。就是原地修改了
这个列表对象。

如果想拷贝 L1，在对 L1 做修改是又不影响到 L2 的时候。就需要拷贝 L1 指向的对象而不是简单的
将变量同 L1 进行同一指向的赋值操作。

共享引用和相等

‘==’ 操作符是测试两个被引用的对象是否有相等的值
‘is’ 操作符是测试对象的同一性，如果两个变量指向同一个对象，会返回 True

虽然两个变量指向对象的值相等，当指向的并不是同一个对象

这个现象是因为 Python 内部有一个小整数池，从 -5~256 当引用小整数时都会在这里引用。
超过这个范围的整数会在内存空间开辟新的空间来保存。所以 is 得到不一样的结果。
